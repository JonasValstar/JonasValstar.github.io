Something that I have always been interested about is random terrain generation. I wanted to try and make my own algorithm at first. To experiment with it and see what problems I would run into.


For generating the trees, I didn’t have an idea of where I could start so I decided, so did some research and found a technique called Poisson Disc Sampling. The Terrain height generation I did want to try myself.

Version 1 - constant distance between points

Version 2 - random distance between points

For generating the trees, I used an algorithm called Poisson Disc Sampling. The system starts with placing a random tree somewhere and creating possible generation points in a circle around it. Then the system checks all those possible points and sees if it can place a tree there without it being to close to an already existing tree. This is repeated until there are no possible points left.


First, I had a constant distance that needed to be between trees. But when making it a random distance it became much less dense and natural looking.

Algorithm 1 - small bumps

Algorithm 2 - large hills (cube visualization)

The first system I made loops over every point in a grid and takes the average height of the generated points around it. Then it would add a random number to the height that would always slightly favour going to the middle.


This resulted in a lot of very small hills, so I tried a different approach. The algorithm started with a small grid (5x5 for example). It would randomise every point on that grid. Then it would double the lengths of the grid (10x10 in this example) and place all the original points on the even numbers. All the odd numbers were set to be near the average of the points around it. This would repeat until the system had a size grid it was happy with.


This resulted in a low number of big hills, but somewhat flat looking. So, for the final version, I combined the first and second algorithm to create a somewhat convincing looking result.
