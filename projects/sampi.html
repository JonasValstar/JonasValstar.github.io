sampi

/

In this project, I wanted to experiment with making an AI opponent for a chess like game. Together with my friend Matthijs, we came up with a new board game for which I would create the AI. This was one of the first projects I did in unity, so it was a great opportunity to learn more about the engine and programming.


I decided it would be a good idea to come up with an algorithm for the AI on my own instead of using an existing algorithm that would likely work better. I did this so that instead of copying an algorithm, I would figure out the solution on my own and get a deeper understanding of how and why other algorithms work.

The first part after coming up with the concept was making it work in Unity. I made a system that detects where the player is clicking to know if they were clicking on their pieces. It would then detect the possible and legal moves they could make with that piece and display it on the board. It would also check if a piece can capture another piece, and if done remove that other piece from the board. It would also keep track of all the positions for the AI to use later.

First version of the AI. It is still slow and makes dumb decisions

Latest version of the AI. It is significantly faster and doesn't make blunders

To determine the best move to make, the AI first loops through all the pieces on the board, determines every possible move and saves all the board states of what would happen if that move was played. It repeats this for a certain number of times (called “Depth”) and then picks the best result.


The best result is determined using the piece positions and some tweakable variables (“Weights”). The algorithm determines the value of all the pieces based on their type and position, the average distance from the start and if the pieces cover each other for attacks. This is done for every board state it gathered, and the highest value is picked from that.


This means that the AI will usually assume the opponent only makes blunders seeing as that results in a better final board state for the AI. To combat this, there are checks to make sure the AI wouldn’t make any blunders, and would always take free pieces. This is still far from perfect, but worked well enough for the time.
