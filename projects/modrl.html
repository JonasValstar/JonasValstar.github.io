In this project, I want to experiment with making something fully alterable by the player. I decided to make a weapon system where the weapons were fully changeable in behaviour by applying new mods. I also wanted to experiment with Node trees, so I made a level tree to unlock new weapons.


I also wanted to make the code I wrote as easily understandable and workable for other developers and adaptable based on things I could add later. For this I tried making something called UML diagrams, to convey how my code works together. I also made UI adapt to whatever items were available to the player.

Mods change the behaviour of a weapon


I made the guns have several mod groups to which the mods would be applied. Each group only accepts mods of that group. Every time a new mod was added or removed; the mods would load in again to make sure the behaviour is correct. The system cleans the weapons behaviour and adds all the equipped mods behaviour into it, along with the weapons base behaviour.


The mods can be setup in a couple of different ways. They can add damage to the bullets the weapons shoots in one of a couple damage types. They can modify a weapons stat, like the delay between firing shots or the maximum amount of ammo. They can add events to existing events like firing the weapons or reloading. Or they can add events to happen on a regular interval. For example, heal every 20 seconds.

Level tree unlocking new weapon

The level tree is made from a list holding structs containing the information that a node needs. Things like the weapon it unlocks. But those structs also have a list in them that holds all the nods that are available after that node is unlocked.


If a node in unlocked, it adds the weapons it has to the player’s inventory

Changeable Mod UI size

Adaptable Level UI layout

The mod UI can be setup by the developer in how many cells wide and high each area is (equipped mods and available mods). It will then automatically calculate the size of each cell and place the mods in these cells.


The level tree UI checks every node in the list and their children and stores them in a different list based on how deep it the node was found. It then sees which depth (row) has the most nodes and adjusts the size of all nods to fit based on that row and the total amount of rows. After that, it goes along every node again and places lines between them and their children.
